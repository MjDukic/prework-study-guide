HTML

  • The head element contains information about the webpage.
  • The meta element contains information about the page that is used by the browser, user wont see information in the head or meta elements.
  • The body element represents the visible content shown to the user.
  • important: html attributes are always defined in the beginning element, never the closing element
  • best practice is to use semantic elements whenever possible, makes html easier to read

CSS

  • css defines the colors, aesthetics, visual layout, and organization
  • the three ways to style a webpage using CSS are inline CSS, internal CSS, and external CSS. external is most commonly used.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • the * is a wild card symbol, declares that all elements on the page will have the style properties, margin and padding, set as 0
  • based on video, its useful to use GitHub copilot for debugging, encourages to comment more

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commants always start with "git"
  • markdown ".md" is a way content filed are formatted in GitHub
  • to make pull/push requests.. navigate to directory we need, git add -A, git commit -m "your message here", git pull origin main, git push origin feature/"the file", then go to GitHub
